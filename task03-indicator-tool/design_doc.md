## 1. 产品概述

### 1.1 一句话定义

一个纯前端的交互式 Web 工具，用户选择股票 → 调节指标参数 → 实时重算并重绘 RSI / MACD / 布林带 / ATR 四项技术指标，辅助投资决策。

### 1.2 核心价值

| 痛点 | 解决方案 |
|------|----------|
| Notebook 中调参数要改代码、重新运行，迭代慢 | 滑块拖动即时重算重绘，秒级反馈 |
| 不同股票要重复跑 Notebook | 股票选择器一键切换，数据预加载 |
| 四个指标分散在不同 cell，难以横向对比 | 2×2 仪表盘 + 信号汇总表统一呈现 |
| 参数敏感性无法直观感知 | 参数滑块 + 实时图表，所见即所得 |

### 1.3 目标用户

- 个人投资者：快速查看技术面信号
- 量化研究者：快速验证指标参数的敏感性
- 金融学习者：直观理解指标参数对图形的影响

### 2.1 数据加载策略

```
用户选择股票 → 前端 fetch 对应 CSV → 解析为 JSON → 缓存到内存
→ 参数变更时从缓存重算指标 → 重绘图表
```

- 首次加载后缓存在浏览器内存，切换参数不重新请求
- 切换股票时才重新 fetch
- 支持未来扩展：从 Tushare API 实时拉取（需后端代理）

## 5. 指标参数设计

### 5.1 RSI — 相对强弱指数

| 参数 | 类型 | 范围 | 默认值 | 步长 | 说明 |
|------|------|------|--------|------|------|
| period | 滑块 | 5 ~ 30 | 14 | 1 | 计算窗口天数 |
| overbought | 滑块 | 60 ~ 90 | 70 | 1 | 超买阈值 |
| oversold | 滑块 | 10 ~ 40 | 30 | 1 | 超卖阈值 |

**重绘触发**: 任一参数变化 → 重算 RSI → 重绘折线 + 参考线 + 填充区域

### 5.2 MACD — 指数平滑异同移动平均线

| 参数 | 类型 | 范围 | 默认值 | 步长 | 说明 |
|------|------|------|--------|------|------|
| fast | 滑块 | 5 ~ 20 | 12 | 1 | 快线 EMA 周期 |
| slow | 滑块 | 15 ~ 40 | 26 | 1 | 慢线 EMA 周期 |
| signal | 滑块 | 5 ~ 15 | 9 | 1 | 信号线 EMA 周期 |

**约束**: fast < slow（UI 自动校验，slow 滑块最低值不低于 fast 当前值）

**重绘触发**: 任一参数变化 → 重算 EMA/DIF/DEA/MACD柱 → 重绘折线 + 柱状图 + 金叉死叉标注

### 5.3 布林带 — Bollinger Bands

| 参数 | 类型 | 范围 | 默认值 | 步长 | 说明 |
|------|------|------|--------|------|------|
| window | 滑块 | 10 ~ 30 | 20 | 1 | 移动平均窗口 |
| num_std | 滑块 | 1.0 ~ 3.0 | 2.0 | 0.1 | 标准差倍数 |

**重绘触发**: 任一参数变化 → 重算中轨/上下轨/带宽/%B → 重绘通道 + 带宽面板

### 5.4 ATR — 平均真实波幅

| 参数 | 类型 | 范围 | 默认值 | 步长 | 说明 |
|------|------|------|--------|------|------|
| period | 滑块 | 5 ~ 30 | 14 | 1 | 计算窗口天数 |
| stop_multiplier | 滑块 | 1.0 ~ 4.0 | 2.0 | 0.1 | 止损倍数 (止损价 = Close - N×ATR) |

**重绘触发**: period 变化 → 重算 TR/ATR → 重绘; stop_multiplier 变化 → 仅重绘止损线

### 5.5 参数预设

| 预设名 | RSI | MACD | 布林带 | ATR | 适用场景 |
|--------|-----|------|--------|-----|----------|
| 标准 | 14, 70, 30 | 12, 26, 9 | 20, 2.0 | 14, 2.0 | 默认参数 |
| 保守 | 21, 75, 25 | 12, 26, 9 | 20, 2.5 | 21, 3.0 | 减少假信号 |
| 激进 | 7, 65, 35 | 5, 20, 5 | 10, 1.5 | 7, 1.5 | 提高灵敏度 |
| 超短 | 5, 80, 20 | 3, 10, 3 | 10, 1.0 | 5, 1.0 | 日内/超短线 |

## 6. 信号判定规则

### 6.1 RSI 信号

| 条件 | 信号 | 显示 |
|------|------|------|
| RSI > overbought | 超买 | 红色标记 |
| RSI < oversold | 超卖 | 绿色标记 |
| oversold ≤ RSI ≤ overbought | 中性 | 灰色 |
| RSI 上穿 oversold (前一日<, 当日≥) | 超卖回升 | ↑ 买入提示 |
| RSI 下穿 overbought (前一日>, 当日≤) | 超买回落 | ↓ 卖出提示 |

### 6.2 MACD 信号

| 条件 | 信号 | 显示 |
|------|------|------|
| DIF 上穿 DEA (前一日<, 当日≥) | 金叉 | ↑ 红色箭头 |
| DIF 下穿 DEA (前一日>, 当日≤) | 死叉 | ↓ 绿色箭头 |
| DIF > 0 且 DIF > DEA | 多头 | 红色柱 |
| DIF < 0 且 DIF < DEA | 空头 | 绿色柱 |
| MACD柱 绝对值连续3日缩小 | 动能衰减 | ⚠ 黄色标记 |

### 6.3 布林带信号

| 条件 | 信号 | 显示 |
|------|------|------|
| %B > 1.0 | 突破上轨 | 红色标记 |
| %B < 0.0 | 跌破下轨 | 绿色标记 |
| 带宽 < 近60日带宽的20分位 | 挤压(Squeeze) | 紫色高亮区间 |
| 带宽 > 近60日带宽的80分位 | 扩张(Expansion) | 橙色标记 |
| 0 < %B < 1 且接近 0.5 | 中轨附近 | 中性 |

### 6.4 ATR 信号

| 条件 | 信号 | 显示 |
|------|------|------|
| ATR > 近60日均值的 1.5 倍 | 波动加剧 | 红色高亮 |
| ATR < 近60日均值的 0.5 倍 | 波动低迷 | 蓝色高亮 |
| 始终显示 | 止损价 = Close - multiplier×ATR | 虚线 |

## 7. 技术架构

### 7.1 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 渲染 | 原生 HTML + CSS Grid/Flexbox | 无框架依赖，单文件部署 |
| 图表 | **ECharts 5** (CDN) | 金融图表能力强，支持蜡烛图、十字光标、dataZoom |
| 计算 | 原生 JavaScript | 指标计算逻辑简单，无需引入 pandas 等库 |
| 数据 | fetch CSV → 解析 → 内存缓存 | 零后端，纯静态部署 |

### 7.2 前端架构

```
┌─────────────────────────────────────────┐
│              UI Layer (HTML/CSS)          │
│  Header | Sidebar | Chart Area | Table   │
├─────────────────────────────────────────┤
│           Interaction Layer (JS)          │
│  事件绑定 → 参数收集 → 触发重算            │
├─────────────────────────────────────────┤
│           Calculation Layer (JS)          │
│  RSI | MACD | Bollinger | ATR 计算引擎    │
├─────────────────────────────────────────┤
│             Data Layer (JS)               │
│  CSV Loader → Parser → Memory Cache      │
├─────────────────────────────────────────┤
│         Data Files (Static CSV)           │
│  688981.SH | 002594.SZ | 600900.SH       │
└─────────────────────────────────────────┘
```

### 7.3 数据流

```
1. 页面加载
   → 默认加载第一只股票的 CSV
   → 解析为 [{date, open, high, low, close, vol}, ...]
   → 存入 stockDataCache[ts_code]

2. 用户切换股票
   → 检查 cache 是否已有该股票数据
   → 若无 → fetch CSV → 解析 → 存入 cache
   → 触发全指标重算 → 重绘所有图表

3. 用户调节参数
   → 收集当前参数值
   → 从 cache 取原始数据
   → 重算对应指标
   → 重绘对应图表 (不影响其他指标)

4. 用户切换日期范围
   → 过滤数据子集
   → 全指标重算 (基于子集)
   → 全图表重绘
```

### 7.4 性能设计

| 场景 | 数据量 | 目标延迟 | 策略 |
|------|--------|----------|------|
| CSV 加载 | ~1600 行 | < 200ms | 首次加载后缓存 |
| 单指标重算 | ~1600 点 | < 50ms | 纯 JS 计算 |
| 单图表重绘 | ~1600 点 | < 200ms | ECharts setOption |
| 全仪表盘重绘 | 4 图表 | < 500ms | 并行 setOption |
| 日期范围过滤 | ~1600→~500 | < 30ms | Array.filter |

## 8. 交互设计细节

### 8.1 参数滑块

- 使用原生 `<input type="range">` + 实时数值显示
- 拖动过程中使用 **debounce 150ms** 触发重算（避免拖动过程中疯狂重绘）
- 松手后立即触发一次精确重算
- 数值显示在滑块右侧，实时更新

### 8.2 图表交互

| 交互 | 行为 |
|------|------|
| 鼠标悬停 | 显示十字光标 + tooltip (日期/OHLC/指标值) |
| 滚轮缩放 | X 轴时间范围缩放 |
| 拖拽平移 | 左右拖拽平移时间轴 |
| 双击 | 重置缩放到全量 |
| 图表联动 | 仪表盘模式下，四图共享 X 轴缩放/平移 |

### 8.3 视觉规范

遵循中国股市惯例（红涨绿跌）：

| 元素 | 颜色 | 色值 |
|------|------|------|
| 涨 / 买入信号 | 红色 | #E24B4A |
| 跌 / 卖出信号 | 绿色 | #639922 |
| 价格线 | 蓝色 | #378ADD |
| 信号线 | 紫色 | #7F77DD |
| 辅助/警告 | 橙色 | #BA7517 |
| 中性/参考 | 灰色 | #999999 |
| 填充区域 | 半透明 | alpha 0.12~0.18 |

### 8.4 响应式

| 断点 | 布局 |
|------|------|
| ≥ 1200px | 左侧栏 + 主区域 2×2 仪表盘 |
| 768~1199px | 顶部控制栏 + 主区域 2×2 仪表盘 |
| < 768px | 顶部控制栏 + 主区域单列堆叠 |
